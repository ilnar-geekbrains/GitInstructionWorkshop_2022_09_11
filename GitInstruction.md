## Работа с удаленными репозиториями.
### Клонирование репозитория
Вы можете создать полную локальную копию репозитория Git из общедоступного проекта, клонировав его командой *git clone ссылка_на_репозиторий*

    git clone ссылка_на_репозиторий

### Отправка данных в удаленный репозиторий
Push - это отправка данных на сервер, в удаленный репозиторий, на github. Перед пушем надо зафиксировать текущие изменения, то есть сделать git commit.  

    git push origin имя_ветки

---
### Загрузки содержимого из удаленного репозитория
Команда git pull используется для извлечения и загрузки содержимого из удаленного репозитория и немедленного обновления локального репозитория этим содержимым. 

    git pull 

---

### Как настроить совместную работу

1. Создать аккаунт на GitHub.com
2. Создать локальный репозиторий
3. “Подружить” ваш локальный и удалённый репозитории.
GitHub при создании нового репозитория подскажет, как это можно сделать
4. Отправить (push) ваш локальный репозиторий в удалённый (на GitHub), при этом, возможно,
вам нужно будет авторизоваться на удалённом репозитории
5. Провести изменения “с другого компьютера”
6. Выкачать (pull) актуальное состояние из удалённого репозитория
